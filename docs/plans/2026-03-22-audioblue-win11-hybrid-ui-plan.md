# AudioBlue Win11 Hybrid UI + Installer Plan

## Summary
- 把当前“最小托盘菜单 MVP”升级为 **Windows 专用混合桌面应用**：保留 Python WinRT 连接后端与托盘常驻能力，新增 **WebView2 承载的可视化控制中心**，UI 采用 **React + TypeScript + Vite + Fluent 2 风格组件/令牌**，整体视觉和交互向 Win11 靠拢。
- 交付两个用户入口：**托盘快速面板** 和 **主界面控制中心**。托盘负责高频操作，主界面负责设备管理、自动连接规则、启动项、通知、诊断、安装与更新入口。
- 自动连接不再局限于“重连上次设备”，而是实现 **优先级规则引擎**：支持收藏设备、优先级顺序、应用启动时自动连接、设备再次出现时自动连接音频、失败回退到下一个候选设备。
- 安装与分发采用 **PyInstaller + Inno Setup** 作为主交付路径，支持开机自启动、静默后台启动、开始菜单与卸载；保留未来补充 **MSIX** 的升级路径，但本轮不实现双分发。

## Key Changes
- **应用架构**
  - 保留现有 Python 后端：`ConnectorService` 继续负责设备枚举、连接、断开、状态变化与后台 worker。
  - 新增 `desktop_host` 层，负责：
    - 托盘图标生命周期
    - 托盘快速面板窗口
    - 主界面窗口
    - Python ↔ WebView2 前端桥接
  - 采用 **WebView2 宿主 + 本地静态前端资源** 的单机混合架构；不引入远程前端服务，不依赖联网。
  - 前端使用 `React + TypeScript + Vite`，组件视觉采用 **Fluent 2 / Win11 风格**；优先使用 Fluent 2 设计令牌与 React 组件体系，必要时允许局部用 Web Components 补充。
  - 状态流分为三层：
    - `connector_service`：设备/连接真实状态
    - `app_state` / `view_model`：前端可消费的 UI 状态
    - `rules_engine`：自动连接、优先级与恢复策略
  - Python 作为单一业务真源；前端只持有派生状态，不直接拥有连接逻辑。

- **UI 结构与视觉**
  - **托盘快速面板**
    - 左键打开，右键仍保留传统上下文菜单作为兜底。
    - 展示当前连接状态、最近设备、快速连接/断开、自动连接开关、打开主界面、打开蓝牙设置、退出。
    - 使用轻量浮层窗口，视觉采用 Acrylic 风格、圆角、阴影、分组卡片和短动效。
  - **主界面控制中心**
    - 使用 Win11 风格布局：左侧导航、右侧内容区、顶部命令栏、Mica 背景与卡片分层。
    - 页面固定为：
      - `概览`：当前状态、最近连接、最近失败、快捷操作
      - `设备`：所有候选设备卡片、状态、连接按钮、收藏、优先级、规则入口
      - `自动化`：规则列表、启用状态、优先级排序、触发方式
      - `设置`：常规、通知、启动、自启动、主题、诊断、关于
    - 不做自定义蓝牙配对流程；新增设备、删除配对、系统输出切换统一深链到 Windows 设置。
  - **Win11 设计细则**
    - 主窗口使用 Mica/Mica Alt，快速面板使用 Acrylic。
    - 图标采用单色基底 + 状态叠层，不使用高频闪烁。
    - 支持浅色/深色/跟随系统。
    - 支持高对比度和键盘导航。
    - 动效只用于面板出现、卡片状态切换和连接 loading，时长控制在约 120–180ms。

- **功能能力扩展**
  - **连接与自动连接**
    - 设备支持以下规则：
      - 仅手动连接
      - 应用启动时自动连接
      - 设备再次出现时自动连接音频
    - 多设备采用优先级队列：
      - 先尝试收藏且启用自动连接的设备
      - 失败后按优先级依次回退
      - 单设备成功连接后终止后续尝试
    - 记录最近成功连接时间、最近失败原因、最近一次自动连接触发来源。
  - **设备管理**
    - 收藏/置顶
    - 忽略设备
    - 仅显示支持音频接入的设备
    - 诊断模式下显示“全部发现设备”
    - 每个设备显示：名称、状态、最近在线、最近连接结果、是否支持自动连接
  - **通知与诊断**
    - 通知策略：静默、仅失败通知、全部通知
    - 错误文案人类可读，不直接暴露底层状态码
    - 导出日志、显示探针结果、打开配置目录、查看当前规则快照
  - **启动与后台**
    - 支持登录 Windows 后自动启动
    - 支持 `--background` 参数，后台启动时直接进入托盘
    - 支持“启动后延迟 N 秒再自动连接”，默认启用短延迟以规避蓝牙栈刚登录时不稳定
    - 仅允许单实例，二次启动激活已有主窗口
  - **安装与升级**
    - `PyInstaller` 继续产出运行目录
    - `Inno Setup` 负责安装、卸载、快捷方式、开机自启动选项、安装后启动
    - 安装器默认提供：
      - 创建开始菜单项
      - 登录 Windows 后启动
      - 安装完成后立即运行
      - 桌面快捷方式默认关闭
    - 本轮只做本地安装器，不做自动更新服务；主界面保留“检查更新”入口占位和未来接口。

- **重要接口与新增模块**
  - `src/audio_blue/connector_service.py`
    - 扩展为可订阅式后端服务，输出统一事件模型而不是零散 dict。
    - 新增连接中、失败原因、候选设备过滤、自动重试友好状态。
  - `src/audio_blue/models.py`
    - 增加前端所需结构：
      - `DeviceCapabilities`
      - `DeviceRule`
      - `ConnectionAttempt`
      - `NotificationPreferences`
      - `StartupPreferences`
      - `UiPreferences`
    - `AppConfig` 扩展为完整设置对象，而不再只有 `reconnect` 与 `last_devices`。
  - 新增 Python 模块：
    - `desktop_host.py`：窗口与宿主管理
    - `app_state.py`：后端事件转 UI 状态
    - `rules_engine.py`：自动连接优先级和触发逻辑
    - `autostart_manager.py`：Windows 开机自启动注册与读取
    - `notification_service.py`：Toast/托盘通知策略
    - `diagnostics.py`：探针、日志、运行状态聚合
    - `single_instance.py`：单实例与窗口激活
  - 新增前端目录：
    - `ui/` 作为 Vite 工程根
    - `ui/src/pages`：概览/设备/自动化/设置
    - `ui/src/components`：设备卡片、状态条、规则编辑器、命令栏、设置分组
    - `ui/src/bridge`：Python JS bridge 封装
    - `ui/src/theme`：Fluent 2 token、自定义品牌色和 Win11 风格变量
  - 新增前后端桥接接口：
    - `get_initial_state()`
    - `refresh_devices()`
    - `connect_device(device_id)`
    - `disconnect_device(device_id)`
    - `update_device_rule(device_id, rule_patch)`
    - `reorder_device_priority(device_ids)`
    - `set_autostart(enabled)`
    - `set_theme(mode)`
    - `set_notification_policy(policy)`
    - `open_bluetooth_settings()`
    - `export_diagnostics()`
  - 事件推送统一为一条总线：
    - `devices_changed`
    - `connection_changed`
    - `connection_failed`
    - `rules_changed`
    - `settings_changed`
    - `diagnostics_changed`

- **实现顺序**
  - 先抽后端状态与规则层，再接主界面壳层，再做托盘可视化，再做自启动和安装器，最后补通知、诊断和 polishing。
  - 托盘右键菜单在新快速面板稳定前保留为降级路径，避免 UI 未完成期间丢失控制能力。
  - 自动连接逻辑在 UI 完成前也应先作为纯后端规则引擎落地并测试，避免界面做完后才发现行为不稳。

## Assumptions
- 采用 **方案 B** 且固定为 **Python 后端 + WebView2 混合前端**；不切换到 WinUI 3 壳层。
- 前端技术栈固定为 **React + TypeScript + Vite**；Fluent 2 作为 UI 设计体系，优先采用 React 组件与设计令牌，不再规划纯原生 Win32 主界面。
- 自动连接策略固定为 **优先级规则模型**，不是“只记住上次设备”或“只支持一台首选设备”。
- 主分发方式固定为 **PyInstaller + Inno Setup**；MSIX 只保留为未来路线，不纳入本轮实现。
- 托盘仍然是常驻主入口，主界面是控制中心；不是每次启动都弹出窗口。
- Windows 蓝牙配对和系统音频输出选择继续委托给系统设置，不在应用内重造。
